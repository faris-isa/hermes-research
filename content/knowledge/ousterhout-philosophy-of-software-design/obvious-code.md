# Obvious Code

> Software should be designed for **ease of reading**, not ease of writing.

## Obvious Code

- Clear names that convey intent
- Related code kept together
- Minimal special cases
- Patterns that match developer expectations

## Non-Obvious Code

- Clever tricks or micro-optimizations
- Overly terse one-liners
- Non-standard patterns requiring special knowledge
- Implicit dependencies

## Consistency

Apply consistent conventions across:
- Variable and method naming
- Code formatting and structure
- Error handling patterns
- Design patterns

Consistency reduces cognitive load — developers do not have to relearn conventions per module.
